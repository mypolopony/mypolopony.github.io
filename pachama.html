1. Graham's Algorithm¶

Briefly, Graham's algorithm suggests to pick an extreme point (here, I've chosen to sort by y, then by x) and then, from that point, analyze the remaining points, determine their angular direction, and try to make right hand turns (or, left hand turns if you're in a former British Commonwealth Country)

def cross(p1, p2, p3):
    '''
    The cross product of two vectors (in this case (p1, p2) and (p2, p3)) is 

    If the value is
        < 0 then we must be going clockwise
        > 0 then we must be going counter-clockwise
        == 0 then oh gosh, we have some colinear points, that requires some more qualification, let's
            just hope we don't run into that problem for the present moment
    '''
    return (p2[0] - p1[0]) * (p3[1] - p1[1]) - (p2[1] - p1[1]) * (p3[0] - p1[0])

def slope(p1, p2):
    '''
    The slope of the line between two points is just rise over run
    '''
    return (p1[1] - p2[1]) / (p1[0] - p2[0])

# Two-factor sort to find the smallest left point and pop it from the list
points = sorted(points, key = lambda p: (p[1], p[0]))
start = points[0]
points.pop(points.index(start))

# Sort points based on the slope, essentially in a circle
points.sort(key=lambda p: (slope(p, start), -p[1], p[0]))

# Add each point to the hull. This requires some memory about the points that have come
# before; essentially, we want to remove previous missteps if we discover that
# we've found a better path. 
#
# You can also think about this in temrs of whether or not your path has, within it,
# a previous point that you assumed to be on the boundary. That is to say, if the next
# step you take creates a polygon that contains the n-2 point, that previosu point is
# not in the boundary. In this way, it is a stepwise "seeking" algorithm
#
# More directly said, if the last 3 points make a clockwise turn, the second to last point is wrong. 

ans = [start]

for p in points:
    ans.append(p)

    # We want to check to make sure that:
    #   1. There's still stuff to do
    #   2. The cross product. criteria are met
    while len(ans) > 2 and cross(ans[-3], ans[-2], ans[-1]) < 0:
        ans.pop(-2)

4. Gift Wrapping Algorithm¶

Personally, I like this idea since it seems oddly intuitive, but I havent had the time to implement it. I think the general idea is to define a circle or sphere, origined at the centroid of all the points, that encompasses all of the points, then 'shrink' it, which would allow you to hit all of the external points in sequence; then you would just stop when. . . all the points are accounted for, either on the boundary or within it. I think that's how that'd work.
